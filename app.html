Why Advanced Profiling? — Real-World Incidents

Standard profiling (null counts, row counts) missed all of these. Statistical metrics catch what basic checks cannot.

Knight Capital (2012)

Lost $440M in 45 minutes from undetected outliers in trading data. Kurtosis + skewness profiling would have caught the errant prices within seconds.

Zillow Offers (2021)

$569M writedown from distribution drift in housing prices. Relative entropy would have flagged the model-vs-reality divergence.

Unity Ad Platform (2022)

$110M lost revenue from "not representative" ingested data. Entropy monitoring would have detected the anomaly before model retraining.

Healthcare EHR (2015-2020)

Patient safety events from silently frozen medication feeds. Value age profiling detects stale timestamps instantly.

Get Started

1. Start OpenMetadata

git clone https://github.com/open-metadata/OpenMetadata.git
cd OpenMetadata/docker/development
docker compose up -d

2. Clone this repo

git clone https://github.com/prasanthcp/extend-profilers-openmetadata.git
cd extend-profilers-openmetadata

3. Configure

Edit config.json with your OM URL, DB credentials, and target tables/schema. Use the form below or download the demo config.

4. Run the profiler

java -jar extend-profilers-openmetadata-0.1.0-SNAPSHOT-all.jar config.json

5. View results

Open output/LatestReport.html or check the Profiler tab in OM UI for custom metrics.

                            Metric Reference
                        

                        
                    
                    

                        

                            

                                
Entropy

                                
Diversity of values. >1.0 = healthy. <1.0 = low diversity, possible value collapse.

                            

                            

                                
Relative Entropy

                                
KL divergence from uniform. <0.5 = balanced. >2.0 = few values dominate.

                            

                            

                                
Kurtosis

                                
Outlier frequency. |v| < 2 = normal. |v| > 5 = heavy outliers.

                            

                            

                                
Skewness

                                
Distribution asymmetry. |v| < 1 = symmetric. |v| > 2 = heavily skewed.

                            

                            

                                
Value Age (hours)

                                
Data freshness. <168h = fresh. >1000h = stale, pipeline may be broken.

                            

                            

                                
Seasonality

                                
Cycle in profile history. >0 = repeating pattern every N runs. 0 = no pattern.
